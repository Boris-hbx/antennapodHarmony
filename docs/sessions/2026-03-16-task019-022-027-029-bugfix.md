## 7. Task-032 BUG-1/3/6: 长按/拖拽手势冲突修复

**用户指令**: 参考 task-032-t3-bugfix.md 修复 BUG-1、BUG-3、BUG-6

### 问题描述
- BUG-1: 剧集列表长按无反应
- BUG-3: 订阅页长按封面无反应
- BUG-6: 队列拖拽和长按无反应

### 第一次修复尝试（失败）

将 `.gesture(LongPressGesture)` 改为 `.priorityGesture(LongPressGesture)`。

**结果**: 仍然无反应。

**原因**: `priorityGesture` 只优先于子组件的**手势**，但子组件的 `.onClick()` 不属于手势系统，它直接拦截 touch 事件。`.swipeAction()` 也会接管触摸事件。

### 第二次修复（成功）

**核心方案**: 移除子组件的大面积 `.onClick()`，将点击和长按统一到 ListItem/GridItem 层级，用 `GestureGroup(GestureMode.Exclusive)` 管理。

#### EpisodeListPage.ets
```diff
- // episodeRow Builder 内的 Row
- .onClick(() => {
-   router.pushUrl({ url: 'pages/EpisodeDetailPage', params: { itemId: item.id } });
- })

  // ListItem 上
- .priorityGesture(
-   LongPressGesture({ repeat: false })
-     .onAction(() => { ... })
- )
+ .gesture(
+   GestureGroup(GestureMode.Exclusive,
+     LongPressGesture({ repeat: false })
+       .onAction(() => {
+         this.selectedEpisode = item;
+         this.showContextMenu = true;
+       }),
+     TapGesture()
+       .onAction(() => {
+         router.pushUrl({ url: 'pages/EpisodeDetailPage', params: { itemId: item.id } });
+       })
+   )
+ )
```

#### SubscriptionsPage.ets
列表模式 + 网格模式两处同样改法：移除内层 `.onClick()`，改为 `GestureGroup(Exclusive, LongPress, Tap)`。

#### QueuePage.ets
1. 移除原 `.bindContextMenu(ResponseType.LongPress)` — 该 API 被 swipeAction + 内层 onClick 拦截
2. 新增 `@State showItemMenu / selectedItem / selectedIndex`
3. 添加 `bindSheet` + `@Builder itemMenuSheet()` + `menuRow()`（与 EpisodeListPage 风格统一）
4. 移除内层 Row 的 `.onClick()`，改为 `GestureGroup(Exclusive, LongPress, Tap)`

### 为什么 GestureGroup(Exclusive) 有效
- `Exclusive` 模式下，框架让多个手势竞争同一组触摸事件
- LongPressGesture 持续按压 >500ms 胜出 → 弹出菜单
- TapGesture 在手指抬起时胜出（未触发 LongPress）→ 导航/多选
- 整个手势识别在同一层级，不存在父子拦截
- 内层小面积按钮（播放/下载/移除）的 `.onClick()` 不受影响

## 8. 经验总结（第二次）

写了 1 条经验：

| 编号 | 类型 | 标题 |
|------|------|------|
| PIT-039 | Pitfall | ListItem 长按被子组件 onClick 和 swipeAction 双重拦截 — priorityGesture 无效，必须用 GestureGroup(Exclusive, LongPress, Tap) |

与 PIT-019 的区别：PIT-019 解决**同一组件**上 onClick + gesture 冲突（用 priorityGesture）；PIT-039 解决**父子组件**之间的冲突（priorityGesture 也不够，必须移除子组件 onClick 并用 GestureGroup 替代）。
